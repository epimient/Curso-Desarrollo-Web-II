Arquitectura

¿Que es MVC?

Modelo Vista Controlador

MVC separa la aplicacion en tres capas: datos (Modelo), presentacion (Vista) y logica de enrutamiento (Controlador).

En español simple Imagina un restaurante: el menu es la Vista (lo que ves), la cocina es el Modelo (donde se prepara la comida) y el mesero es el Controlador (recibe el pedido y lo lleva a cocina).

FastAPI no te obliga a usar MVC, pero su diseno lo fomenta.

Primer endpoint

Hola FastAPI: GET basico

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def raiz():
    return {"mensaje": "Hola mundo"}

@app.get("/") registra una funcion que responde a peticiones GET en la raiz. Todo lo que devuelve la funcion se convierte en JSON automaticamente.

Path parameters

Rutas dinamicas con path parameters

@app.get("/cursos/{curso_id}")
def obtener_curso(curso_id: int):
    return {
        "id": curso_id,
        "nombre": f"Curso {curso_id}"
    }

Las llaves {curso_id} capturan el valor de la URL. FastAPI lo convierte al tipo que declares (int, str, etc.).

Tip Si pides /cursos/42, FastAPI convierte "42" al entero 42 automaticamente. Si pones texto, devuelve error 422.

Query parameters

Parametros opcionales en la URL

@app.get("/cursos/")
def listar_cursos(skip: int = 0, limit: int = 10):
    return {
        "skip": skip,
        "limit": limit,
        "cursos": []
    }

Los query parameters son lo que va despues de ? en la URL. FastAPI los toma del nombre del parametro de la funcion.

Type hints

Python type hints: la base de FastAPI

def saludar(nombre: str, edad: int) -> str:
    return f"Hola {nombre}, tienes {edad} anos"

FastAPI usa los type hints para:

  • Validar automaticamente los tipos de datos.
  • Generar documentacion OpenAPI.
  • Detectar errores en tiempo de desarrollo.
En español simple Los type hints le dicen a FastAPI "esto espero recibir". Si llega otra cosa, lo rechaza antes de que toque tu codigo.

Pydantic

Pydantic BaseModel: validacion automatica

from pydantic import BaseModel

class Curso(BaseModel):
    nombre: str
    creditos: int
    activo: bool = True

Con Pydantic defines la forma de tus datos. FastAPI valida automaticamente que el JSON que llega coincida con el modelo.

Dato Pydantic es la libreria de validacion de datos mas usada en Python. FastAPI la incluye como dependencia central.

POST

Crear recursos con POST y Pydantic

@app.post("/cursos", status_code=201)
def crear_curso(curso: Curso):
    return {
        "id": 1,
        **curso.dict()
    }

FastAPI recibe el JSON, lo valida contra el modelo Curso y te lo entrega ya validado. Si falta un campo obligatorio, devuelve 422.

Codigos HTTP

Codigos de estado en endpoints

200 201 404 422

FastAPI te permite definir el codigo HTTP con status_code. El valor default es 200.

@app.post("/cursos", status_code=201)
@app.get("/cursos/{id}", status_code=200)

Asi NO vs Asi SI

Antipatrones comunes

❌ Asi NO: todo en una funcion

@app.get("/{tipo}/{id}")
def todo_en_uno(tipo, id):
    if tipo == "cursos":
        ...
    elif tipo == "users":
        ...

Un solo endpoint que hace de todo. Ilegible e imposible de mantener.

✅ Asi SI: un endpoint por recurso

@app.get("/cursos/{id}")
def obtener_curso(id: int): ...

@app.get("/users/{id}")
def obtener_user(id: int): ...

Cada recurso tiene su propio endpoint. Claro y mantenible.

response_model

Controlar la respuesta con response_model

class CursoOut(BaseModel):
    id: int
    nombre: str
    creditos: int

@app.get("/cursos/{id}", response_model=CursoOut)
def obtener_curso(id: int):
    curso = buscar_en_bd(id)
    return curso  # se filtra automaticamente

response_model filtra los campos que se devuelven. Sirve para no exponer datos sensibles (ej: contraseñas).

Cierre

Lo que debe quedar claro

  • FastAPI usa Python type hints para validacion y documentacion.
  • Los endpoints se organizan con decoradores @app.get(), @app.post(), etc.
  • Pydantic models = contrato de datos.
  • HTTP tiene verbos y codigos con significado estandar.
