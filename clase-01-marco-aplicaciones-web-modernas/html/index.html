Pregunta guia

¿Que ocurre cuando presionas un boton?

Cliente HTTP Servidor Logica Respuesta

En español simple Cada vez que usas una app web, tu navegador (cliente) le pide datos a un servidor usando el idioma HTTP. El servidor procesa la peticion y te devuelve una respuesta.

Concepto base

Una aplicacion web no es solo una pagina bonita

  • Recibe acciones del usuario.
  • Procesa reglas de negocio.
  • Consume o expone APIs.
  • Protege datos y permisos.
  • Persiste informacion.
Recordatorio El 90% del codigo de una app web no se ve: esta en el servidor, no en el navegador.

Cada verbo HTTP tiene un significado. El servidor entiende que accion quieres realizar segun el verbo que uses.

HTTP

Codigos de estado: el servidor responde

CodigoSignificadoEjemplo
200OKGET exitoso
201CreadoPOST exitoso
404No encontradoRuta inexistente
500Error internoFallo del servidor

Los codigos HTTP son como semaforos: te dicen si todo esta bien (verde), si algo falta (amarillo) o si algo fallo (rojo).

API

Una API es un contrato

GET /cursos

[
  {
    "id": 1,
    "nombre": "Desarrollo Web II",
    "creditos": 3
  }
]

El cliente sabe que pedir; el servidor sabe que responder. Eso es una API: un acuerdo sobre como comunicarse.

En español simple API es como el menu de un restaurante. Tu pides por el numero (endpoint) y el cocinero (servidor) te prepara lo que pide el menu (respuesta).

Frameworks

Un framework reduce caos repetitivo

Rutas Validacion Seguridad Documentacion Middleware Pruebas

Sin framework, cada API empieza desde cero. Con framework, tienes las herramientas listas y solo escribes las reglas de tu negocio.

FastAPI

¿Por que FastAPI?

  • Construido sobre Python moderno (type hints).
  • Validacion automatica con Pydantic.
  • Documentacion interactiva (OpenAPI + Swagger).
  • Rendimiento comparable a Node.js o Go.
  • Curva de aprendizaje suave.
Dato clave FastAPI es el framework que mas crece en el ecosistema Python. Empresas como Microsoft, Uber y Netflix lo usan.

Mini-lab

Consumir una API publica con Python

import requests

url = "https://api.github.com/repos/tiangolo/fastapi"
response = requests.get(url)
data = response.json()

print(data["name"])
print(data["stargazers_count"])

En 6 lineas consumiste una API real. Esto es lo que hace tu navegador cada vez que usas una app web.
