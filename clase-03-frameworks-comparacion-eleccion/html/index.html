Frameworks: comparacion y eleccion

Analisis comparativo de frameworks web Python y criterios para elegir el adecuado.

Ing. Eduardo Pimienta

Objetivos

Al finalizar esta clase podras

  • Comparar FastAPI, Flask, Django y Bottle segun criterios tecnicos.
  • Justificar por que FastAPI es la opcion para este curso.
  • Identificar el ecosistema de herramientas que acompanan a FastAPI.
  • Elegir un framework segun el tipo de proyecto.

Problema

¿Que problema resuelven los frameworks?

  • Sin framework: escribes HTTP, routing, validacion, serializacion desde cero.
  • Con framework: te preocupas solo de la logica de tu negocio.
  • El framework estandariza: rutas, respuestas, errores, seguridad.
En español simple Un framework es como una cocina industrial: tiene horno, hornallas, heladera y utensilios. Tu solo te preocupas de cocinar.

Criterios de eleccion

¿Como elegir un framework?

  • Tipo de proyecto: API pura? Frontend incluido? Microservicio?
  • Rendimiento: Esperas miles o millones de requests?
  • Equipo: Junior? Senior? Necesitas documentacion automatica?
  • Tiempo: Lanzamiento rapido o proyecto a largo plazo?
  • Ecosistema: Que librerias necesitas? BD, auth, testing?

FastAPI

¿Por que FastAPI para este curso?

Python moderno Type hints Pydantic OpenAPI Async Testing facil

FastAPI combina lo mejor de Flask (simplicidad) y Django (potencia) con rendimiento de alto nivel.

OpenAPI

Documentacion automatica: el as bajo la manga

Tu codigo FastAPI OpenAPI JSON Swagger UI 
# Sin escribir ni una linea extra:
http://localhost:8000/docs   # Swagger UI
http://localhost:8000/redoc  # ReDoc

OpenAPI es el estandar de la industria para documentar APIs. FastAPI lo genera automaticamente.

Async

Async / Await: rendimiento nativo

@app.get("/async")
async def leer_datos():
    data = await db.fetch("SELECT ...")
    return data

FastAPI soporta async de forma nativa. Mientras una request espera a la BD, otra puede avanzar. Esto multiplica la capacidad del servidor.

Asi NO vs Asi SI

Errores al elegir framework

❌ Elegir por moda

"Todo el mundo usa Django, asi que lo usamos." Sin evaluar si es la herramienta correcta.

✅ Elegir por criterios

"Necesitamos una API rapida con validacion automatica y documentacion. FastAPI es la mejor opcion."

Troubleshooting

Errores de concepto

ErrorRealidad
"FastAPI es solo para async"Soporta sync y async, tu eliges.
"Flask es mas rapido porque es mas simple"FastAPI es mas rapido gracias a Starlette.
"Django es el unico profesional"Profesional es elegir la herramienta correcta.
"Necesito Django si quiero ORM"SQLAlchemy funciona con cualquier framework.

Cierre

Lo que debe quedar claro

  • Cada framework tiene un perfil: rapido, completo, minimalista.
  • FastAPI ofrece el mejor equilibrio para APIs modernas.
  • La decision debe basarse en criterios, no en moda.
  • FastAPI tiene un ecosistema solido: Pydantic, SQLAlchemy, Alembic.
