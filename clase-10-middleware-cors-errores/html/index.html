Middleware

Middleware personalizado con @app.middleware

import time
from fastapi import Request

@app.middleware("http")
async def add_process_time(
    request: Request, call_next
):
    inicio = time.perf_counter()
    response = await call_next(request)
    tiempo = time.perf_counter() - inicio
    response.headers["X-Process-Time"] = f"{tiempo:.4f}"
    return response
ParteQue hace
@app.middleware("http")Registra la funcion como middleware
call_nextPasa la request al siguiente en la cadena
await call_next(request)Obligatorio: sin esto la request nunca llega al endpoint

Asi NO vs Asi SI

Middleware: el error mas comun

❌ Sin call_next

@app.middleware("http")
async def malo(r, call_next):
    print("Request")
    # FALTA: await call_next(r)
    # El cliente se queda
    # esperando para siempre

✅ Con call_next

@app.middleware("http")
async def bueno(r, call_next):
    print("Antes")
    resp = await call_next(r)
    print("Despues")
    return resp

CORS

CORS: el bar con lista de invitados

En español simple CORS es un mecanismo del navegador. Cuando tu frontend en localhost:5173 habla con tu backend en localhost:8000, el navegador dice: "Son origenes diferentes. ¿El backend acepta visitas de este origen?"
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

Sin CORS: el frontend recibe error en consola. Con CORS: el backend dice explicitamente "si, este origen puede acceder".

Preflight

Preflight OPTIONS: la llamada de prueba

Frontend OPTIONS /courses Backend 200 + headers CORS DELETE real

Antes de enviar un DELETE (o request con headers custom), el navegador hace una pregunta timida con OPTIONS. Si el backend responde bien, procede con la request real.

Tip FastAPI + CORSMiddleware maneja el preflight automaticamente. No necesitas hacer nada extra.

Errores

Manejo global de errores: central de alarmas

❌ Error 422 default

{
  "detail": [
    {"loc": ["body","name"],
     "msg": "field required",
     "type": "value_error"}
  ]
}

Confuso para el usuario.

✅ Error 422 personalizado

{
  "error": true,
  "codigo": 422,
  "mensaje": "Datos invalidos",
  "detalles": [
    "body -> name: field required"
  ]
}

Claro y consistente.

Exception handlers

Personalizar errores con @app.exception_handler

from fastapi.exceptions import RequestValidationError

@app.exception_handler(RequestValidationError)
async def validation_handler(
    request: Request, exc: RequestValidationError
):
    errores = []
    for error in exc.errors():
        campo = " -> ".join(str(l) for l in error["loc"])
        errores.append(f"{campo}: {error['msg']}")

    return JSONResponse(
        status_code=422,
        content={
            "error": True,
            "codigo": 422,
            "mensaje": "Datos invalidos",
            "detalles": errores,
        },
    )

Excepciones propias

Crear excepciones personalizadas

class NotFoundException(HTTPException):
    def __init__(self, recurso: str, id: int):
        super().__init__(
            status_code=404,
            detail=f"{recurso} con id {id} no encontrado",
        )

class ForbiddenException(HTTPException):
    def __init__(self, msg="No tienes permisos"):
        super().__init__(status_code=403, detail=msg)

# Uso:
raise NotFoundException("Course", course_id)

En lugar de escribir raise HTTPException(404, "Course not found") cada vez, usas tu propia excepcion con el mensaje incluido.

Lifespan

Lifespan: startup y shutdown

from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):
    # ── STARTUP ──
    print("Iniciando API...")
    Base.metadata.create_all(bind=engine)
    yield
    # ── SHUTDOWN ──
    print("API detenida.")

app = FastAPI(lifespan=lifespan)
En español simple El lifespan define que pasa cuando la app arranca (crear tablas, conectar servicios) y cuando se apaga (cerrar conexiones). Es como abrir y cerrar el local cada dia.

TrustedHost

TrustedHostMiddleware: seguridad extra

from fastapi.middleware.trustedhost import TrustedHostMiddleware

app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=["localhost", "127.0.0.1", ".miapp.com"],
)

Protege contra ataques de "Host Header Injection". Solo acepta requests de hosts en la lista.

Orden

Orden recomendado de middlewares

# 1. Seguridad
app.add_middleware(TrustedHostMiddleware, ...)
# 2. CORS
app.add_middleware(CORSMiddleware, ...)
# 3. Personalizados
app.add_middleware(LoggingMiddleware)

# 4. Error handlers
register_error_handlers(app)

# 5. Routers
app.include_router(...)

El orden importa. El primer middleware agregado es el primero en recibir la request.

Troubleshooting

Errores comunes con middleware y CORS

ErrorSolucion
Sin header Access-Control-Allow-OriginFalta CORSMiddleware o allow_origins mal configurado
Invalid Host HeaderTrustedHostMiddleware no incluye ese host
Middleware no se ejecutaFalta await call_next(request)
Errores 422 con formato defaultNo registraste el handler de RequestValidationError
CORS con allow_origins=["*"] y credentials=TrueNo se puede. Usa origenes explicitos.
Lifespan no funcionaFalta yield dentro del context manager

Cierre

Lo que debe quedar claro

  • Middleware = filtro global para todas las rutas.
  • CORS es del navegador, no de la API. Postman no lo tiene.
  • Los exception handlers centralizan el formato de errores.
  • Lifespan reemplaza a @app.on_event (deprecado).
  • El orden de middlewares importa.
