Modelos Pydantic y validacion

Field(), validadores personalizados, modelos anidados y manejo de errores de validacion.

Ing. Eduardo Pimienta

Objetivos

Al finalizar esta clase podras

  • Usar Field() para agregar reglas de validacion a los modelos.
  • Escribir validadores personalizados con @field_validator.
  • Crear modelos anidados (un modelo dentro de otro).
  • Interpretar y solucionar el error 422.

Problema

Sin validacion, datos basura entran a la API

# ❌ Sin validacion: esto es aceptado
{ "nombre": "", "creditos": 99 }

# ✅ Con validacion: rechazado
# nombre debe tener >= 3 caracteres
# creditos debe ser entre 1 y 6

La validacion es la primera linea de defensa de tu API.

Analogia

Pydantic es un filtro de agua

Datos sucios Filtro Pydantic Datos limpios

El filtro revisa: tipo correcto?, rango valido?, campos requeridos presentes?. Si algo falla, corta el paso (error 422).

En español simple Pydantic es el portero de la discoteca: revisa que tengas la edad, el DNI y la entrada antes de dejarte pasar.

Field()

Field(): reglas de validacion

from pydantic import BaseModel, Field

class Curso(BaseModel):
    nombre: str = Field(min_length=3, max_length=80)
    creditos: int = Field(ge=1, le=6)
    email: str = Field(pattern=r"^[\w.]+@[\w.]+\.\w+$")
ParametroQue hace
min_lengthTamaño minimo del string
max_lengthTamaño maximo del string
geMayor o igual (greater or equal)
leMenor o igual (less or equal)
patternExpresion regular que debe cumplir
defaultValor por defecto si no se envia

@field_validator

Validadores personalizados

from pydantic import field_validator

class Usuario(BaseModel):
    email: str
    password: str

    @field_validator("email")
    @classmethod
    def email_debe_ser_institucional(cls, v):
        if not v.endswith("@universidad.edu"):
            raise ValueError(
                "Debe ser email institucional"
            )
        return v

El validador recibe el valor, lo revisa y lo devuelve (o lanza error).

Modelos anidados

Un modelo dentro de otro

class Autor(BaseModel):
    nombre: str
    email: str

class Libro(BaseModel):
    titulo: str
    autor: Autor       # ← modelo anidado

class Biblioteca(BaseModel):
    nombre: str
    libros: list[Libro]  # ← lista de modelos

Pydantic valida automaticamente todos los niveles de anidacion.

model_config

Configuracion del modelo

from pydantic import ConfigDict

class Curso(BaseModel):
    model_config = ConfigDict(
        extra="forbid",   # ❌ rechaza campos extra
        frozen=True,      # 🔒 inmutable
    )

    nombre: str
    creditos: int
extra="forbid"
Rechaza campos no declarados
frozen=True
Impide modificar el objeto despues de creado

Error 422

Entendiendo el error 422

{
  "detail": [
    {
      "loc": ["body", "creditos"],
      "msg": "Input should be a valid integer",
      "type": "int_parsing"
    }
  ]
}
CampoSignificado
locDonde esta el error (body, path, query)
msgMensaje del error en ingles
typeTipo de error (valor faltante, tipo incorrecto)

El error 422 es tu aliado: te dice exactamente que falta o esta mal.

Asi NO vs Asi SI

Validacion basica vs avanzada

❌ Sin validacion extra

class Curso(BaseModel):
    nombre: str
    # Acepta strings vacios
    # Acepta creditos negativos

✅ Con Field() y validadores

class Curso(BaseModel):
    nombre: str = Field(
        min_length=3,
        max_length=80
    )
    creditos: int = Field(ge=1, le=6)

    @field_validator("nombre")
    def no_espacios(cls, v):
        if not v.strip():
            raise ValueError(...)
        return v

Codigo completo

Ejemplo: Libro con autor anidado

class AutorCreate(BaseModel):
    nombre: str = Field(min_length=3)
    email: str

    @field_validator("email")
    @classmethod
    def validar_email(cls, v):
        if "@" not in v:
            raise ValueError("Email invalido")
        return v

class LibroCreate(BaseModel):
    titulo: str = Field(min_length=1)
    autor: AutorCreate
    anio: int = Field(ge=1900, le=2030)

Troubleshooting

Errores comunes con Pydantic

ErrorSolucion
"field required"Falta el campo en el JSON. Revisa el modelo.
"extra fields not permitted"Enviaste campos extras. Usa extra="forbid" o eliminalos.
"Input should be a valid integer"Enviaste texto donde se espera numero.
"String should have at least X characters"El texto es muy corto. Aumenta o revisa el input.
ValueError en validador propioTu @field_validator lanzo una excepcion. Revisa la logica.
TypeError: Pydantic v1 vs v2En v2 se usa @field_validator (no @validator).

Cierre

Lo que debe quedar claro

  • Field() define reglas basicas: rangos, longitudes, patrones.
  • @field_validator crea reglas personalizadas (logica condicional).
  • Los modelos anidados permiten estructuras complejas.
  • El error 422 es descriptivo: usalo para depurar.
