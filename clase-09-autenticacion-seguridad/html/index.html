Autenticacion y seguridad con JWT

Hash de contraseñas, tokens JWT, proteccion de endpoints y manejo de roles.

Problema

Sin autenticacion, cualquiera puede hacer cualquier cosa

❌ Sin proteccion

DELETE /courses/1
# Cualquiera puede borrar
# Sin token, sin permiso

Inseguro. No sabes quien llama a la API.

✅ Con proteccion

DELETE /courses/1
Authorization: Bearer <token>
# Solo usuarios autenticados
# Solo admins pueden borrar

Sabes quien llama y si tiene permiso.

Analogia

Tienda sin puerta vs Tienda con seguridad

Sin autenticacion
Tienda sin puerta: cualquiera entra, agarra productos, modifica precios, borra inventario.
Con autenticacion
Tienda con puerta + recepcionista + credencial: muestras tu carnet (token), te identifican y segun tu rol te dejan hacer ciertas cosas.

Hash

Hash de contraseñas: la licuadora

En español simple Guardar contraseñas en texto plano es como dejar la combinacion de tu caja fuerte escrita en un post-it. Un hash es como pasar la contraseña por una licuadora: obtienes un pure irreconocible, y no puedes volver a la fruta original.
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"])

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain, hashed) -> bool:
    return pwd_context.verify(plain, hashed)

JWT

JWT: el carnet de identidad digital

Login JWT Almacena Envia Verifica 
from jose import jwt
from datetime import datetime, timedelta

def create_token(data: dict) -> str:
    payload = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=30)
    payload.update({"exp": expire})
    return jwt.encode(payload, SECRET_KEY, algorithm="HS256")

El JWT contiene: quien eres (sub), cuando expira (exp), y que rol tienes (role).

Flujo

Flujo completo: registro → login → endpoint protegido

1. POST /users/register
   { "username": "ana", "password": "123456" }
   → Hash de password, guarda en BD

2. POST /token (form data)
   username=ana&password=123456
   → Verifica hash, genera JWT

3. GET /users/me
   Authorization: Bearer <jwt>
   → Decodifica JWT, devuelve usuario

OAuth2

OAuth2PasswordBearer: el seguridad de la puerta

from fastapi.security import OAuth2PasswordBearer

oauth2_scheme = OAuth2PasswordBearer(
    tokenUrl="/token"
)

def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: Session = Depends(get_db),
) -> User:
    payload = decode_token(token)
    user = db.query(User).filter(
        User.id == int(payload["sub"])
    ).first()
    return user

OAuth2PasswordBearer le dice a FastAPI: "esta ruta necesita un token valido en el header Authorization".

Endpoint /token

Desglose del endpoint /token

@router.post("/token", response_model=Token)
def login(form_data: OAuth2PasswordRequestForm = Depends(),
          db: Session = Depends(get_db)):

    user = db.query(User).filter(
        User.username == form_data.username).first()

    if not user or not verify_password(
        form_data.password, user.hashed_password):
        raise HTTPException(401)

    token = create_access_token(
        data={"sub": str(user.id), "role": user.role})

    return Token(access_token=token, token_type="bearer")

Recibe user+pass → verifica hash → genera token → devuelve JWT.

Proteger endpoint

Proteger un endpoint con get_current_user

@router.get("/users/me", response_model=UserResponse)
def read_users_me(
    current_user: User = Depends(get_current_user),
):
    return current_user

@router.delete("/courses/{id}")
def delete_course(
    id: int,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    if current_user.role != "admin":
        raise HTTPException(403, "Not enough permissions")
    # ... borrar curso

Si no hay token valido → 401. Si hay token pero sin permisos → 403.

Roles

Roles: student vs admin

RolPuede hacer
studentVer cursos, ver su perfil
adminTodo lo anterior + crear/borrar cursos, cambiar roles
 
if current_user.role != "admin":
    raise HTTPException(
        status_code=403,
        detail="Accion solo para administradores"
    )

El rol se guarda en el JWT y se verifica en cada endpoint protegido.

Asi NO vs Asi SI

Endpoint abierto vs protegido

❌ Endpoint abierto

@router.delete("/courses/{id}")
def delete(id: int):
    # Cualquiera borra
    # Sin restriccion

✅ Endpoint protegido

@router.delete("/courses/{id}")
def delete(id: int,
  current=Depends(get_current_user)):
    if current.role != "admin":
        raise HTTPException(403)
    # Solo admin borra

Troubleshooting

Errores comunes con autenticacion

ErrorSolucion
401 UnauthorizedToken invalido, expirado, o no enviado. Revisa el header Authorization.
403 ForbiddenToken valido pero sin permisos. El usuario no es admin.
422 en /tokenLos datos no tienen el formato correcto. Revisa username y password.
JWTError al decodificarEl token esta corrupto o fue manipulado.
Token expiradoEl token tiene tiempo limitado. Genera uno nuevo haciendo login.

Cierre

Lo que debe quedar claro

  • Nunca guardar contraseñas en texto plano. Siempre hash con bcrypt.
  • JWT = carnet digital firmado por el servidor.
  • get_current_user protege endpoints.
  • 401 = no identificado. 403 = identificado pero sin permiso.
