Persistencia de datos con SQLAlchemy

Bases de datos, ORM, sesiones y operaciones CRUD con SQLAlchemy 2.0.

Ing. Eduardo Pimienta

Objetivos

Al finalizar esta clase podras

  • Conectar FastAPI a una base de datos SQLite usando SQLAlchemy.
  • Definir modelos SQLAlchemy y crear las tablas automaticamente.
  • Usar Session para operaciones CRUD: crear, leer, actualizar, borrar.
  • Inyectar la sesion de BD como dependencia en FastAPI.

Problema

Los datos en memoria se pierden al reiniciar

❌ Sin persistencia

# Lista en memoria
cursos = []
# Al reiniciar el servidor
# todo se borra 😢

Los datos viven solo mientras la app esta corriendo.

✅ Con BD

# Datos en disco
cursor.execute(
  "SELECT * FROM cursos"
)
# Sobreviven a reinicios

Los datos persisten aunque apagues el servidor.

Analogia

Pizarra vs Cuaderno

Pizarra (memoria)
Escribes, ves, pero al borrar la pizarra se pierde todo. Rapido pero temporal.
Cuaderno (BD)
Escribes, cierras el cuaderno, lo abres al otro dia y los datos siguen ahi.

La BD es tu cuaderno: guarda los datos para siempre (o hasta que los borres explicitamente).

ORM

ORM: el traductor Python ↔ SQL

Python ORM SQL
En español simple Un ORM te permite hablar con la BD usando objetos de Python en lugar de escribir SQL. En vez de SELECT * FROM cursos, escribes db.query(Curso).all().

database.py

database.py: conexion a la BD

from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, DeclarativeBase

engine = create_engine("sqlite:///./cursos.db",
    connect_args={"check_same_thread": False})

SessionLocal = sessionmaker(
    autocommit=False, autoflush=False, bind=engine)

class Base(DeclarativeBase):
    pass

Modelo SQLAlchemy

Modelo: la clase que representa una tabla

from sqlalchemy import Column, Integer, String, Boolean
from app.database import Base

class Course(Base):
    __tablename__ = "courses"

    id = Column(Integer, primary_key=True)
    name = Column(String(80), nullable=False)
    credits = Column(Integer, nullable=False)
    active = Column(Boolean, default=True)

Cada atributo del modelo es una columna de la tabla.

get_db

Dependencia get_db: sesion por request

from app.database import SessionLocal

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
En español simple Cada vez que llega un request, abrimos una conexion a la BD. Cuando termina, la cerramos. Asi evitamos fugas de conexiones.

CRUD

Operaciones CRUD en el service

def list_courses(db: Session):
    return db.query(Course).all()

def get_course(db: Session, id: int):
    return db.query(Course).filter(
        Course.id == id
    ).first()

def create_course(db: Session, data):
    course = Course(**data.dict())
    db.add(course)
    db.commit()
    db.refresh(course)
    return course

El service recibe db: Session como parametro, inyectado por FastAPI.

SQLite vs PostgreSQL

¿Que BD usar?

CriterioSQLitePostgreSQL
InstalacionNinguna (incluida)Requiere servidor
ConcurrenciaLimitadaAlta
ProduccionNo recomendadoEstandar
DesarrolloPerfectoPosible

SQLite es ideal para desarrollo y aprendizaje. En produccion usa PostgreSQL.

Asi NO vs Asi SI

SQL plano vs ORM

❌ SQL plano en el codigo

cursor.execute(
  "SELECT * FROM courses WHERE id = ?",
  (id,)
)
# Fragil, error-prone,
# dificil de mantener

✅ ORM con SQLAlchemy

db.query(Course).filter(
  Course.id == id
).first()
# Seguro, legible,
# portable entre BD

Troubleshooting

Errores comunes con SQLAlchemy

ErrorSolucion
sqlalchemy.exc.IntegrityErrorViolacion de unicidad o FK. Revisa los datos.
DetachedInstanceErrorUsaste un objeto fuera de la sesion. Usa db.refresh().
La tabla no se creaFalta Base.metadata.create_all(bind=engine)
sqlite3.ProgrammingErrorLa BD esta bloqueada por otra operacion. Usa check_same_thread=False
AttributeError: 'NoneType' has no attribute 'name'La query devolvio None. Revisa si el registro existe.

Lo que debe quedar claro

  • CRUD se hace con: query, filter, add, commit, refresh.
