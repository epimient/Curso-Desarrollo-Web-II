Configuracion

Variables de entorno

La configuracion NO va en el codigo. Va en el ambiente.

# app/core/config.py
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    database_url: str = "sqlite:///./cursos.db"
    secret_key: str = "changeme"
    environment: str = "development"
    model_config = SettingsConfigDict(env_file=".env")
VariableDesarrolloProduccion

Infraestructura

Supabase: PostgreSQL gratis

  • 500 MB de almacenamiento — gratis
  • No expira (vs Render Postgres que expira a 30 dias)
  • SSL incluido
  • Conexion desde cualquier servidor
# Connection string de Supabase
postgresql://postgres:password@db.xxx.supabase.co:5432/postgres

En Supabase, las tablas se crean automaticamente con Base.metadata.create_all() en el lifespan.

Despliegue

Render: crear Web Service

  1. Sube tu codigo a GitHub
  2. Render.com → New + → Web Service
  3. Conecta tu repositorio
  4. Configura:
    • Runtime: Python 3
    • Build Command: pip install -r requirements.txt
    • Start Command: fastapi run app/main.py --port 10000
    • Plan: Free
  5. Agrega variables de entorno (DATABASE_URL, SECRET_KEY, etc.)
  6. Deploy → espera ~3 minutos

Despliegue

Variables de entorno en Render

Estos valores se configuran en el dashboard de Render (no en el codigo):

VariableValor
DATABASE_URLpostgresql://... (de Supabase)
SECRET_KEYgenerada con openssl
ENVIRONMENTproduction
CORS_ORIGINS["https://tu-app.onrender.com"]
ALLOWED_HOSTS[".onrender.com"]

Nunca subas valores reales a Git. Solo usa variables de entorno.

Despliegue

Supabase: crear BD

  1. supabase.com → New project
  2. Name: api-academica-db
  3. Crea una contrasena segura (guardala)
  4. Espera ~2 min a que se cree
  5. Settings → Database → Connection string
  6. Copia la URI y reemplaza [YOUR-PASSWORD]
  7. Pega esa URI en DATABASE_URL de Render

Supabase PostgreSQL no expira. 500MB gratis para siempre.

CI/CD

Despliegues automaticos

Render escucha cambios en GitHub. Cada git push = nuevo deploy.

git add .
git commit -m "Fix: validacion de email"
git push origin main
# Render detecta el cambio, reconstruye y despliega

(Opcional) GitHub Actions para ejecutar tests antes del deploy:

name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install -r requirements.txt
      - run: pytest

Buenas practicas

Secretos y CORS en produccion

❌ Mal

  • secret_key = "123456" en el codigo
  • database_url hardcodeado
  • .env subido a Git
  • CORS permitiendo cualquier origen

✅ Bien

  • SECRET_KEY en variables de entorno
  • DATABASE_URL desde env
  • .env en .gitignore
  • CORS solo con origenes permitidos

Actividad

Desplegar tu API

  1. Migra config.py a pydantic-settings con env_file.
  2. Actualiza database.py (sin connect_args SQLite).
  3. Crea proyecto en Supabase, copia DATABASE_URL.
  4. Sube a GitHub.
  5. Crea Web Service en Render (Runtime: Python 3).
  6. Configura DATABASE_URL, SECRET_KEY, ENVIRONMENT.
  7. Verifica: curl https://tu-api.onrender.com/health/.

Cierre

Lo que debe quedar claro

  • pydantic-settings separa configuracion del codigo.
  • PostgreSQL (Supabase) reemplaza a SQLite en produccion.
  • Render aloja tu web service gratis con runtime Python nativo.
  • No necesitas Docker — Render maneja pip install + fastapi run.
  • Las variables de entorno protegen secretos y permiten cambiar config sin modificar codigo.
  • Nunca hardcodees claves, URLs o passwords.
